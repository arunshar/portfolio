TrajPrompt — Explained

One line: type a plain-English question ("ships drifting suspiciously near pipelines last March") and get matching ship trajectories on a map, by training a CLIP-style model that aligns AIS trajectories with text, then confirming candidates with TGARD rendezvous detection and a SAM 2 look at the Sentinel-2 imagery.

Honest status up front. Real and tested (6/6): the trajectory-CLIP encoder + its InfoNCE contrastive loss, and the TGARD group-rendezvous algorithm (find_rendezvous) with pairwise haversine. Stubbed: the SAM 2 / Sentinel-2 chip puller (Sam2ChipPipeline returns zeros, the Planetary-Computer + SAM 2 call is the "production would…" path). Scaffold: the trained traj-CLIP checkpoint and the "AIS-text pairs dataset." In short: "the trajectory-text contrastive encoder and the rendezvous detector are real and tested; the visual-confirmation stage and the trained checkpoint are the next step."

1. What it is and the gap it fills

Maritime analysts still query AIS with SQL filters ("speed < 3 kn AND region = …"). That can't express fuzzy intent like "loitering suspiciously near infrastructure." CLIP solved this for images (search photos by text); TrajPrompt does the same for ship trajectories: learn a shared embedding space where a trajectory and its natural-language description sit close together, so you can retrieve trajectories by typing a sentence. Then it adds two checks: TGARD finds the actual rendezvous events, and SAM 2 confirms a vessel is visibly there in Sentinel-2.

3. Module-by-module walkthrough

src/trajprompt/traj_clip.py — the IP. TrajCLIPEncoder: a small Transformer over a fixed-window AIS trajectory (each step = (lon, lat, sog, cog, distance-to-coast, dwell)), with learned positional embeddings, mean-pooled to a 256-d L2-normalized embedding. trajclip_loss: the symmetric InfoNCE (the exact CLIP objective), trajectory→text and text→trajectory cross-entropy against the diagonal (matching pairs), temperature 0.07. This is what makes "search trajectories by sentence" possible.

src/trajprompt/tgard.py — the group-rendezvous variant of TGARD (the trajprompt port; darkvessel has the gap-feasibility variant). haversine_pairwise computes an N×M great-circle distance matrix. find_rendezvous(points, …): buckets AIS points into 1-minute windows, finds pairs of distinct MMSIs within distance_threshold_m (default 500 m), accumulates contiguous co-location, and emits a RendezvousCandidate when the dwell exceeds min_dwell_seconds (default 10 min), with an optional low-speed gate (so convoys at speed aren't flagged). Output: who met whom, where, for how long, and how close.

src/trajprompt/sam2_chip.py — Sam2ChipPipeline, the visual confirmation. Stub: chip_for returns a zero chip + null mask; the docstring lays out the production path (STAC search on Microsoft Planetary Computer → 256×256 crop at the AIS coordinate → SAM 2 with a "ship" prompt → confidence). confirm retries a ±window-days range to find a clear-sky chip.

src/trajprompt/data/{ais.py, sentinel2.py} — AIS loaders and the Sentinel-2 fetch. tests — 6 tests: haversine correctness, rendezvous detection on synthetic dwell, contrastive-loss alignment (matching pairs score higher), traj-CLIP L2-norm, chip-stub shape.

4. The core idea, from scratch

CLIP, ported to trajectories. CLIP trains two encoders (image, text) so that a true image-caption pair has high cosine similarity and mismatched pairs have low similarity, via InfoNCE over a batch (each item's positive is its own pair; everything else is a negative). TrajPrompt swaps the image encoder for a trajectory Transformer and trains the same loss against sentence embeddings of AIS-track descriptions. After training, a free-text query embeds into the shared space and you retrieve the nearest trajectories, open-vocabulary search, no fixed schema.

Why a Transformer for trajectories? A trajectory is an ordered sequence with long-range structure (a loiter pattern spans many timesteps). Self-attention captures that better than hand-crafted features or an RNN, and the positional embedding encodes step order.

6. What I'd do next

  1. Build a small AIS-text pair set and actually train traj-CLIP, then show one real text→trajectory retrieval. That single retrieval makes the whole thing concrete.
  2. Wire the real SAM 2 chip puller (Planetary Computer STAC) so visual confirmation is live.
  3. Chain it end to end: prompt → traj-CLIP retrieval → TGARD filter → SAM 2 confirm, one dark-vessel case top to bottom.
  4. Evaluate retrieval (recall@k on held-out text queries) so "open-vocabulary search" has a number.
